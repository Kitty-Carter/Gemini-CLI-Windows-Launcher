# Gemini CLI 使用偏好

## 语言与系统

- 默认使用中文回答。
- 我的系统是 Windows。
- 给命令时优先使用 PowerShell。
- 路径中如果有空格，请帮我自动加引号。
- 解释问题时，先给结论，再给步骤。

## 安全规则

- 修改文件前，先说明准备修改什么。
- 不要自动执行危险命令。
- 危险命令包括但不限于：删除文件、清空目录、格式化、修改系统目录、修改注册表、修改全局环境变量、提交或推送 Git、上传密钥。
- 如果命令可能影响系统、账号、密钥、网络代理或重要文件，必须先提醒我确认。
- 不要在 C:\Windows、C:\Program Files、C:\Program Files (x86) 里工作。

## 编程偏好

- 代码尽量简单、清晰、可运行。
- 先给最小可行方案，再考虑优化。
- 遇到报错时，请先判断最可能原因，再给排查步骤。
- 如果要创建文件，请告诉我文件名和位置。
- 如果要运行命令，请解释这条命令做什么。

## 我的使用习惯

- 我经常访问中国大陆地区网站，不要建议我长期使用全局代理。
- Gemini CLI 可以走代理，但不要污染整个 Windows 或 PowerShell 环境。
- 我希望配置尽量集中放在 C:\DIY\Gemini CLI。
